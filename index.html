О курсовой работе

Данная работа посвящена исследованию современных методов шифрования данных, применяемых в веб-проектах. В эпоху цифровой трансформации вопросы информационной безопасности становятся критически важными для любого веб-приложения.

В работе рассматриваются как симметричные, так и асимметричные алгоритмы шифрования, методы хеширования, а также современные протоколы защиты данных при передаче по сети.

Методы шифрования данных

Симметричное шифрование

Использует один ключ для шифрования и дешифрования данных. Быстрое и эффективное для больших объемов данных.

  • AES (Advanced Encryption Standard)
  • DES (Data Encryption Standard)
  • 3DES (Triple DES)

Применение: шифрование локальных данных, защита файлов

Асимметричное шифрование

Использует пару ключей: открытый для шифрования и закрытый для дешифрования. Обеспечивает безопасный обмен ключами.

  • RSA (Rivest-Shamir-Adleman)
  • ECC (Elliptic Curve Cryptography)
  • ElGamal

Применение: SSL/TLS, цифровые подписи

Хеширование

Преобразование данных в уникальную строку фиксированной длины. Необратимое преобразование для хранения паролей.

  • SHA-256, SHA-3
  • bcrypt
  • Argon2

Применение: хранение паролей, проверка целостности

Выводы

В современных веб-проектах оптимальным подходом является комбинация методов: асимметричное шифрование для безопасного обмена ключами, симметричное - для шифрования данных, и хеширование - для защиты паролей. Протоколы типа TLS используют гибридный подход, сочетая преимущества разных методов.

Реализация в веб-проектах

HTTPS и TLS/SSL

Современный стандарт защиты данных при передаче по сети. Использует гибридное шифрование:

  • Асимметричное шифрование для обмена ключами
  • Симметричное шифрование для передачи данных
  • Цифровые сертификаты для аутентификации

Важно: всегда использовать актуальные версии протоколов (TLS 1.2/1.3)

Хранение паролей

Правильные практики хранения учетных данных:

  • Всегда использовать хеширование (никогда не хранить в открытом виде)
  • Применять "соль" (salt) для защиты от rainbow-таблиц
  • Использовать адаптивные функции (bcrypt, Argon2)

Рекомендация: минимальная стоимость фактора для bcrypt - 12

Пример кода: шифрование в JavaScript 

// Пример хеширования с использованием Argon2
async function hashWithArgon2(password) {
    // Подключаем библиотеку argon2-browser
    const argon2 = await import('https://cdn.jsdelivr.net/npm/argon2-browser@1.18.0/dist/argon2-bundled.min.js');
    
    // Генерируем случайную соль
    const salt = window.crypto.getRandomValues(new Uint8Array(16));
    
    // Хешируем пароль с параметрами
    const result = await argon2.hash({
        pass: password,
        salt: salt,
        time: 3,       // Количество итераций
        mem: 65536,    // Используемая память в KiB
        hashLen: 32,   // Длина хеша в байтах
        parallelism: 1 // Количество потоков
    });
    
    return result.encoded;
}

// Пример использования
const password = "mySecretPassword123";
hashWithArgon2(password).then(hash => {
    console.log(`Argon2 hash: ${hash}`);
});

Как это работает?

Демонстрация использует Web Crypto API для реального шифрования данных в браузере. Все операции выполняются локально, данные не отправляются на сервер. Для шифрования файлов используется потоковая обработка с минимальным использованием памяти.
