Ключевые Возможности

Полностью Клиентское

Все операции шифрования и дешифрования происходят непосредственно в вашем браузере, обеспечивая максимальную конфиденциальность. Данные никогда не покидают ваше устройство.

Использование Web Crypto API

Приложение использует стандартизированный Web Crypto API, предоставляемый современными браузерами, для безопасных и быстрых криптографических операций.

Разнообразие Алгоритмов

Демонстрация включает поддержку популярных алгоритмов, таких как AES-GCM для симметричного шифрования и RSA-OAEP для асимметричного.

Шифрование Текста

Легко шифруйте и дешифруйте текстовые сообщения, получая наглядное представление о работе криптографии.

Шифрование Файлов

Возможность шифровать и дешифровать файлы любого размера с помощью потоковой обработки, оптимизированной для больших объемов данных.

О Проекте

Этот проект является курсовой работой по дисциплине "Информационная безопасность" на тему "Методы шифрования данных в web-проектах". Цель проекта — наглядно продемонстрировать принципы работы современных криптографических алгоритмов, используемых в веб-приложениях.

Основное внимание уделяется практическому применению Web Crypto API, который позволяет безопасно выполнять криптографические операции непосредственно в браузере пользователя, без передачи конфиденциальных данных на сервер.

Приложение позволяет тестировать шифрование и дешифрование текстовых данных, а также файлов, используя алгоритмы AES-GCM (для симметричного шифрования) и RSA-OAEP (для асимметричного шифрования). Также предусмотрена генерация и импорт ключей для RSA.

Проект направлен на повышение осведомленности о важности криптографии в веб-разработке и демонстрацию того, как современные веб-технологии позволяют создавать безопасные решения.

Используемые технологии:

  • HTML5 & CSS3
  • JavaScript (ES6+)
  • Web Crypto API
  • Tailwind CSS (для стилизации)
  • Webpack (для сборки модулей, если используется)

Как это работает?

Демонстрация использует Web Crypto API для реального шифрования данных в браузере. Все операции выполняются локально, данные не отправляются на сервер. Для шифрования файлов используется потоковая обработка с минимальным использованием памяти.
